BACKEND

Бэкенд-сервис на базе Django и Django Rest Framework (DRF) выступает в роли центрального ядра системы, где сосредоточена вся бизнес-логика, управление базой данных и административная панель, полностью асинхронный. Он спроектирован так, чтобы обслуживать одновременно два разных клиента: веб-интерфейс на React и Telegram-бота, предоставляя им единый REST API для работы с товарами, заказами и пользователями. Благодаря интеграции с PostgreSQL 15, бэкенд обеспечивает надежное хранение данных и высокую скорость обработки сложных запросов. В архитектуре проекта бэкенд намеренно изолирован внутри внутренней сети Docker и не имеет прямого доступа извне, что защищает его от большинства сетевых атак, так как все входящие запросы проходят через строгую фильтрацию в Nginx.
Административная панель бэкенда кастомизирована для удобного управления контентом, позволяя модераторам загружать изображения товаров и управлять рассылками в реальном времени, при этом статические файлы и медиа-контент эффективно делегируются на внешний Nginx для достижения максимальной производительности.

БОТ

Сервис Телеграм-бота выступает в роли удобного интерфейса для конечных пользователей, позволяя им взаимодействовать с системой без использования браузера. Он написан на фреймворке aiogram и работает как самостоятельный процесс внутри Docker-сети. Бот не имеет прямого доступа к базе данных PostgreSQL, соблюдая принципы чистой архитектуры: все данные о товарах, профилях пользователей и статусах заказов он получает через внутренние HTTP-запросы к Django API по адресу api:8000. Это гарантирует, что бизнес-логика едина и для фронтенда на Реакте, и для мобильного интерфейса в Телеграме.
Для обеспечения безопасности бот использует Docker Secrets, считывая токен и ключи из смонтированных файлов в оперативной памяти, что исключает утечку данных через логи или переменные окружения. Имеет два воркера для исполнения задач уведомления админов о новом заказе и пользователя о смене статуса. ПОлучает сообщения от бекенда через Redis. Так же для локальной разработки бот получает media через смонтированный том

Его главная роль — обеспечить пользователю быстрый и отзывчивый интерфейс для просмотра каталога товаров, управления корзиной и оформления заказов с визуальными эффектами, которые трудно реализовать в стандартном интерфейсе мессенджера.
Сейчас фронт не интегрированн в бота и по сути не является tg webapp. Сделано так для экономии времени на настройке тунеля и удобной разработки. Имеет все фукнции для интеграции в tg.

ЗАПУСК
1. клонирование репозитория
2. копирование папки secrets в корень проекта
3. изменение - VITE_USER_ID=${USER_ID:-.....} на свой id. При повторной смене id нужно пересобрать образ
4. docker compose up
5. по адресу http://127.0.0.1:8001/admin заходим в админку (username = admin, password = 1111)
6. добавляем категории о подкатегории
7. добавляем товары, связываем ТОЛЬКО с подкатегориями для корректной работы (в тз не было указано как именно нужно отображать товары в боте, так что по моей логике они отображаются по подкатегориям). Так же товарам можно назначить не более 1 картинки так как inline кнопки получилось привязать только к одному сообщению
8. добавляем бота в админ чат и вписываем его id в globalconfig. даем боту админку
9. добавляем бота как админа в группы, подписку на которые будем проверять. Добавляем эти группы в subscriptions
